# Requirements

This folder contains raw confirmed requirements and open questions.

## Files

- [requirements-plan.md](requirements-plan.md): confirmed requirements and pending questions captured from user conversations.
- [non-functional-requirements.md](non-functional-requirements.md): confirmed non-functional requirements and quality attribute boundaries.

## Update Rule

When the user confirms, corrects, or rejects a requirement, update [requirements-plan.md](requirements-plan.md) first, then propagate the change to the relevant structured documents.
