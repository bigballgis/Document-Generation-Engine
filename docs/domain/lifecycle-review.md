## 模板生命周期审查

| 主题 | 已确认内容 | 待确认设计议题 |
| --- | --- | --- |
| 状态集合 | 模板状态包括草稿、测试中、测试通过、待审批、待发布、已发布、已停用、已废弃；v1 不新增发布前或发布后中间状态。 | 无。 |
| 协作待办 | v1 协作通知采用站内待办和状态提示；提交测试、测试不通过或回到草稿、提交审批、审批不通过或回到草稿、审批通过进入待发布均生成对应角色队列或整改待办；待办按模板所属组和角色队列分配，不默认指定个人，不授予额外权限。 | 不外发邮件或即时通讯消息；不新增生命周期状态。 |
| 测试前置 | 模板必须经过测试人员测试；提交测试只能从草稿状态或测试通过状态发起，提交测试后进入测试中；模板发布候选必须关联一组或多组用于测试生成的测试数据集，至少包含一组必选测试样例，并保留逐样例测试生成记录、批量测试摘要和覆盖率摘要。 | 无。 |
| 测试判定 | 正常测试判定由测试人员执行；分组管理员只在异常处理或补救场景中例外干预；测试判定需要记录结构化结果和测试意见，意见采用受控表单模板，不复用结构化富文本编辑器；测试通过需要确认测试证据摘要、批量测试摘要、覆盖率摘要、生成预览摘要和保真警告摘要，可填写补充说明；测试不通过必须填写原因分类、影响范围和修复建议；所有必选测试样例必须测试通过且无未解决阻断项，保真警告需要完成摘要查看确认；模板综合覆盖率和关键维度覆盖率低于配置阈值时不得测试通过；分组管理员例外干预时必须填写原因、二次确认并添加单独审计标记。 | 无。 |
| 测试不通过 | 测试不通过后回到草稿；原测试记录作为历史保留；再次提交发布候选时必须完整重新提交测试和审批，生成新一轮测试记录和审批记录。 | 无。 |
| 审批前置 | 模板必须经过审批人员审批；审批为一级审批；测试通过后进入测试通过状态，不自动进入待审批；测试通过状态下，需要由具备提交审批权限的角色手动提交审批，提交审批后进入待审批；提交审批和审批判定需要能查看测试记录、批量测试摘要、覆盖率摘要、生成预览、最终产物引用、预览对比摘要、变更差异摘要和发布前检查清单结果。 | 无。 |
| 审批判定 | 正常审批判定由审批人员执行；分组管理员只在异常处理或补救场景中例外干预；审批判定需要记录结构化结果和审批意见，意见采用受控表单模板，不复用结构化富文本编辑器；审批通过必须填写理由摘要并确认关键证据摘要；审批不通过必须包含退回原因分类、影响范围和整改要求，并关联测试记录、变更差异摘要和发布前检查清单摘要；分组管理员例外干预时必须填写原因、二次确认并添加单独审计标记。 | 无。 |
| 审批不通过 | 审批不通过后回到草稿；原审批记录作为历史保留；再次提交发布候选时必须完整重新提交测试和审批，生成新一轮测试记录和审批记录。 | 无。 |
| 发布 | 审批通过后进入待发布，不直接发布；待发布状态下由具备发布模板权限的角色手动发布，发布权限以权限矩阵为准；发布前必须执行发布前检查清单，存在阻断项时不得发布；发布版本号采用语义化版本，发布时由模板编排人员或管理员选择版本变更级别；发布前必须展示发布摘要并二次确认，发布摘要至少包含发布版本号、变更差异摘要、批量测试摘要、覆盖率摘要、最终产物引用、预览对比摘要、发布前检查清单结果、API 契约摘要和影响范围；发布前检查发现阻断项、必选样例未全部通过或覆盖率低于配置阈值时不新增发布失败状态，模板保持待发布但不得发布，阻断项记录在发布前检查清单中；若修复需要修改模板内容、依赖版本、测试数据集或渲染配置，必须回到草稿或创建新的开发版本，并重新执行测试、审批和发布流程。 | API 契约和调用侧配置在 API 管理中确认。 |
| 超时升级 | 测试待办、审批待办、待发布待办和整改待办支持自动超时升级；超时阈值采用全局默认和组级覆盖；超时只生成对应分组管理员的站内升级待办和状态提示。 | 不自动通过、不自动拒绝、不自动退回、不改变模板状态，不产生代理审批。 |
| 停用 | 模板停用后所有版本都停用；版本级别也支持停用；单个发布版本停用只影响该版本，其他未停用版本仍可按模板级 API 管理配置调用；模板和单个发布版本均允许从已停用状态恢复；恢复需要影响预览、二次确认和审计；模板恢复后，模板下未停用且未废弃的发布版本按模板级 API 管理配置重新进入可调用候选范围；单个发布版本恢复后，仅该发布版本重新进入可调用候选范围，仍受模板状态和 API 管理配置约束。 | 无。 |
| 废弃 | 已废弃表示永久下线，不可恢复；只保留历史和审计；模板废弃前必须先停用且无可调用发布版本；废弃需要二次确认、填写原因、影响预览并记录审计，不强制主动通知调用方。 | 无。 |
| 导入生产 | 导入生产后从草稿阶段重新走流程，不能直接导出为最终发布版本；遇到已有相同模板 ID 时，保留模板 ID，并在目标环境创建新的开发版本；模板发布版本号在后续发布时再选择；不重新生成模板 ID 或 API 地址。 | 无。 |

## 母版审核审查

| 主题 | 已确认内容 | 待确认设计议题 |
| --- | --- | --- |
| 母版生命周期 | 母版不需要类似模板的完整生命周期，也不需要可用/停用状态；母版需要轻量审核状态，状态集合为草稿、待审核、审核通过、审核不通过。 | 无。 |
| 提交审核 | 母版设计人员和分组管理员可按权限提交母版审核；提交审核前必须执行锚点完整性校验并填写变更说明；锚点完整性校验失败时不得提交审核，变更说明进入审核摘要和审计记录。 | 无。 |
| 审核执行 | 母版审核由管理员执行；全局管理员可审核全部母版，分组管理员可审核被授权组范围内母版；母版审核通过后才允许被新建或更新模板引用；草稿、待审核和审核不通过的母版不得被模板引用；审核不通过后回到草稿，并保留审核记录。 | 无。 |
| 母版变更 | 母版更新后不自动影响已有模板；母版变更需要审计；母版变更需要影响分析，影响分析至少包含引用模板清单和重新测试提示。 | 无。 |

## 条款或内容模块生命周期审查

| 主题 | 已确认内容 | 待确认设计议题 |
| --- | --- | --- |
| 状态集合 | 条款或内容模块版本状态包括草稿、待审批、已批准、已停用、已废弃；审批不通过后回到草稿。 | 无。 |
| 独立审批 | 条款或内容模块 v1 采用独立版本审批生命周期；每个模块版本必须独立审批；未批准、已停用或已废弃的版本不得被新的模板发布候选引用。 | 无。 |
| 维护角色 | 不新增专门维护角色；全局管理员、分组管理员、母版设计人员和模板编排人员可按范围创建、编辑草稿、创建新版本并提交审批；审批人员按分组审批；测试人员只在测试材料和生成预览中查看。 | 无。 |
| 共享范围 | 默认按所属组隔离；全局管理员和分组管理员可配置授权组共享范围；模板只能引用同组或已授权共享范围内的模块。 | 无。 |
| 版本锁定 | 模板发布版本必须锁定引用的具体已批准模块版本；模块新版本发布后不自动影响已发布模板；模板升级引用新模块版本时必须执行影响分析，并重新经过测试、审批和发布流程。 | 无。 |
| 运行期影响 | 条款或内容模块停用或废弃后，已发布且已锁定该模块版本的模板仍按发布时锁定内容生成；模块停用或废弃只阻止后续新的模板发布候选引用该模块版本；如果业务需要立即停止使用包含问题模块的已发布模板，必须通过停用对应模板或发布版本来阻断生成和 API 调用。 | 无。 |
| 管理员治理动作 | 条款或内容模块停用、恢复和废弃仅由全局管理员和分组管理员执行；执行前必须进行影响分析、二次确认并记录审计；停用或废弃前的影响分析必须覆盖引用模板、引用发布版本、default 路由影响、近期调用摘要、是否需要停用模板或发布版本，以及可替代模块版本建议。 | 无。 |

## 推荐下一步

1. 模板生命周期、母版审核和条款或内容模块生命周期暂无剩余待确认状态问题。
