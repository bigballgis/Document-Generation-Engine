## 体验审查结论

当前文档对后台治理、权限、API、生命周期和审计描述较完整；模板可验证性基线已确认，覆盖测试数据集、生成预览、变更差异摘要、发布前检查清单、阻断发布规则和 DOCX/PDF 保真警告项。模板创作与渲染边界已确认：母版主要作为版式容器，正文由结构化内容、条款或内容模块、表格组件、变量和规则组合生成，富文本采用受控结构化模型，DOCX/PDF 保真采用阻断项与警告项分级。v1 结构化富文本采用保守节点矩阵，并明确不支持自由 Word/HTML 式排版；Word 或 HTML 粘贴内容必须清洗为受控节点。条款或内容模块已确认独立版本审批、授权共享、模板发布版本锁定和管理员治理动作。结构化片段与母版样式映射已确认通过母版样式目录、命名样式和白名单有限直接格式控制。Word/HTML 粘贴清洗已确认转换范围、阻断条件、清洗摘要和取消/撤销能力。信贷客户经理正式生成入口已确认由上游业务系统承接，文档生成平台 v1 不提供正式生成门户。模板低代码编辑主流程已确认采用分步向导，熟悉工具栏外观必须受控，变量和规则通过面板与可视化配置器维护，条款或内容模块通过检索和显式版本选择引用。保真警告展示、预览权威边界、并排对比体验、多组测试样例、批量测试和覆盖率门禁已确认。当前最大产品体验风险转为测试/审批意见文案、协作通知、API 管理交互和 API 调用方契约查看体验。

## 已确认体验基线

- 模板发布候选需要测试数据集、生成预览、变更差异摘要和发布前检查清单。
- 测试人员和审批人员需要能查看生成预览、变更差异摘要、测试生成记录和发布前检查清单结果。
- 同一模板或发布候选支持多组命名测试样例；样例可标记场景名称、必选或可选、覆盖标签和数据集版本。
- v1 支持对发布候选选择多组测试样例执行批量测试生成，并展示逐样例结果、批量测试摘要、样例覆盖率和模板综合覆盖率。
- 覆盖率至少覆盖变量取值与必填字段、条件/循环/规则分支、锚点/章节、表格等受控组件和 DOCX/PDF 输出格式；模板综合覆盖率和关键维度覆盖率低于配置阈值时阻断发布。
- 所有必选测试样例必须测试通过且无未解决阻断项；可选样例进入批量测试和覆盖率证据，但除非被配置为必选或影响覆盖率阈值，不单独阻断发布。
- 编辑态快速预览只用于即时反馈，不作为测试、审批或发布权威证据；测试、审批和发布以最终 DOCX/PDF 渲染链路产生的预览产物和最终产物引用为准。
- 平台提供预览与最终 DOCX/PDF 产物的并排对比视图，差异定位到页、锚点、章节或组件，并按风险进入保真警告或发布阻断项。
- 测试、审批和发布摘要按输出策略锁定最终 DOCX/PDF 产物引用、预览记录引用、`testDataSetId`、`variablesHash`、`renderProfileVersion`、警告摘要、阻断项摘要、对比摘要和非敏感摘要。
- 发布前检查发现阻断项时，模板不得发布。
- 后台治理与管理界面支持 `REDBC` 红色主题和 `GREENBC` 绿色主题运行时切换，并通过共享主题令牌保持跨页面一致性。
- 后台治理与管理界面主语言为英文，页面默认采用白色背景、经典 OA 桌面优先布局和更舒展的大气排版，不采用过度紧凑的小气布局。
- 顶部全局头部品牌区需要随主题切换同步展示 `REDBC` 或 `GREENBC` 官方 logo；logo 保持官方比例、清晰留白和稳定位置，不通过放大品牌色块制造视觉噪声。
- 测试判定和审批判定需要结构化意见；意见模板采用受控表单字段加可选补充说明，不复用结构化富文本编辑器。
- 测试通过需要确认测试证据摘要、批量测试摘要、覆盖率摘要、生成预览摘要和保真警告摘要，可填写补充说明；测试不通过必须填写原因分类、影响范围和修复建议。
- 审批通过必须填写理由摘要并确认关键证据摘要；审批不通过必须填写退回原因分类、影响范围和整改要求。
- 测试和审批风险提示采用系统默认文案，并允许全局管理员维护全局默认、分组管理员在被授权组范围内维护覆盖文案和原因分类；风险提示至少覆盖未解决阻断项、保真警告、覆盖率低于阈值、预览对比差异、API 契约或影响范围变化和管理员例外干预。
- v1 协作通知采用站内待办和状态提示，不外发邮件或即时通讯消息；提交测试、测试不通过或回到草稿、提交审批、审批不通过或回到草稿、审批通过进入待发布均生成对应角色队列或整改待办。
- 协作待办按模板所属组和角色队列分配，不默认指定个人，不授予额外权限；测试待办、审批待办、待发布待办和整改待办支持自动超时升级。
- 自动超时升级阈值采用全局默认和组级覆盖；超时只生成对应分组管理员的站内升级待办和状态提示，不自动通过、不自动拒绝、不自动退回、不改变模板状态，也不产生代理审批。
- 发布前需要展示发布摘要并二次确认。
- 分组管理员例外干预测试或审批判定时，需要强制原因、二次确认和单独审计标记。
- 母版提交审核前需要锚点完整性校验和变更说明；母版变更影响分析需要包含引用模板清单和重新测试提示。
- 生成预览、变更差异摘要、发布前检查清单和审批摘要的审计或可追溯记录不得保存敏感明文；涉及敏感内容时按审计脱敏规则处理。
- DOCX/PDF 保真警告项采用稳定 warningCode、影响位置摘要、检测结果摘要和处理建议；警告进入测试生成记录、审批材料、发布前检查清单、发布摘要、审计摘要和 API 生成成功响应。
- 保真警告在测试、审批和发布材料中采用摘要条加筛选列表展示；支持按 `warningCode`、锚点/章节/组件位置、预览/DOCX/PDF 产物和已查看状态筛选。
- 测试通过、审批通过和发布前二次确认必须确认已查看保真警告摘要；不要求逐条解决，警告数量不自动阻断发布。
- 后台 API 契约页提供调用方视图，展示授权模板的页面计算契约版本对比、错误码说明、调用示例、可调用版本列表、API 策略摘要、调用方自身 API 凭证非敏感状态、警告码目录、字段含义、示例、文件流响应头说明和授权范围内的非敏感调用结果警告摘要；契约版本对比覆盖请求 Schema、响应 Schema、错误码、API 策略、路由/default 目标、可调用版本和示例，只展示非敏感摘要；v1 不建设独立开发者门户，不提供 API 凭证自助管理或完整审计明细。
- 信贷客户经理正式业务出信由上游业务系统通过动态 API 触发；平台后台手工生成仅用于模板测试、审批预览和管理员治理排查。
- 模板低代码编辑采用分步向导主流程；编辑器可提供熟悉工具栏外观，但只暴露已确认受控节点、母版样式目录和白名单有限直接格式。
- 变量通过 Schema 面板插入为不可拆分变量节点；条件和循环规则通过可视化配置器维护，并结合样例数据即时校验。
- 条款或内容模块通过检索和显式版本选择引用；发布版本锁定具体模块版本，升级引用必须执行影响预览并重新测试审批。
- 登录起点角色旅程首波范围已确认覆盖 `GLOBAL_ADMIN`、`GROUP_ADMIN`、`TEMPLATE_AUTHOR`，并采用“登录 -> 角色落地页 -> 首个关键任务”的链路结构。
- 登录范围本迭代保持现有本地认证假设，不新增 SSO/OIDC 交互流程。
- 新旧导航并行迁移由特性开关控制；开关关闭时保持既有导航行为，避免切换期认知混乱。
- 路由可见性首波基线已确认：全局治理首页仅全局管理员可见；分组治理首页对全局/分组管理员可见；模板编排首页对三类首波角色均可见；API 管理与审计入口延续现有权限边界。
- 已登录用户访问无权限路由时，体验基线为前端阻断并展示统一无权访问反馈，同时不泄露未授权页面数据。
- 首要体验成效指标方向已确认：降低角色关键任务完成时间。

## 友好度原则候选

- 将复杂权限和生命周期状态转化为用户能理解的操作状态和下一步行动。
- 在模板发布前提供模板内容检查清单，覆盖变量、规则、锚点、测试和审批。
- 在 API 管理中提供调用侧检查清单，覆盖 API 凭证、AD Group 授权、输出方式、批量上限、动态加密能力、API 契约和受影响发布版本。
- API 管理配置采用模板级 API 管理页，使用配置域导航 + 详情区；详情区展示当前配置摘要、候选配置编辑区、字段提示、影响预览、硬阻断和警告、保存确认动作、当前 `policyVersion`、最近更新时间、最近操作人和审计入口。
- API 管理配置各配置域采用固定控件和内联提示：AD Group 授权使用可搜索选择器，输出方式使用格式和模式勾选，批量上限使用同步/异步数值输入，DOCX/PDF 动态加密使用启用开关和能力项选择，default 路径目标发布版本使用发布版本选择器。
- API 管理配置各配置域独立保存，操作动线为编辑候选配置、执行影响预览、处理硬阻断或确认警告、管理员确认立即生效；候选配置变更后必须重新执行影响预览。
- API 管理硬阻断和警告文案采用原因、影响、处理建议的固定结构；硬阻断不可继续保存，警告需要管理员确认后继续，并提示会立即生效和记录审计。
- 对生成失败给出业务可读错误，例如缺少必填参数、锚点缺失、版本停用、无模板授权。
- 对低代码规则提供可视化配置、示例数据和即时校验，减少隐藏错误。
- 对结构化富文本提供受控编辑、变量 chip、编号层级、条款片段引用、粘贴清洗和 DOCX/PDF 预览反馈，避免把普通富文本 HTML 当作生成契约。
- 对 100 页级复杂信函，优先使用版式母版、结构化章节、条款/内容模块和表格组件组合，而不是维护一个巨大 DOCX 母版。
- 对审批和测试提供差异摘要，降低人工检查成本。
- 对 API 调用方提供稳定的契约入口、页面计算的非敏感契约版本对比、示例请求/响应、错误码解释、可调用版本列表、API 策略摘要和凭证非敏感状态。

## 待确认产品体验问题

- 角色关键任务完成时间的量化阈值、采样窗口和分环境目标值。
- `TEMPLATE_AUTHOR` 与现有“母版设计人员”在登录落地页导航上的最终融合策略。
- 无权访问反馈页的文案与按钮动作细节（返回上一页、回到首页、联系管理员）。
