## 审查状态

本审查只确认以下事实：

- 已确认平台目标是企业级金融信函低代码文档生成平台。
- 已确认母版为 DOCX，母版通过锚点提供基础框架。
- 已确认模板编排支持锚点内容、模板变量、条件/循环规则、测试生成、审批和发布。
- 已确认锚点内容类型包含富文本段落、表格、条款片段、签章/盖章占位、二维码/条码和附件清单。
- 已确认 DOCX/PDF 输出和 API 管理侧动态加密。

以下内容已确认作为 v1 创作与渲染边界：母版主要作为版式容器和章节级锚点载体；模板编排采用结构化编辑方向；富文本源数据采用受控结构化模型；条款片段升级为可复用、可版本化、可审批的条款或内容模块；DOCX/PDF 预览和渲染保真采用阻断项与警告项分级。

以下内容已进一步确认为 v1 细节：结构化富文本采用保守节点矩阵；v1 明确不支持自由 Word/HTML 式排版；Word 或 HTML 粘贴内容必须清洗并转换为受控结构化节点；严重渲染风险阻断发布。

条款或内容模块生命周期与权限已确认：v1 采用独立版本审批生命周期，复用现有角色体系，默认同组隔离并支持授权共享，发布模板锁定具体已批准模块版本，停用、恢复和废弃由管理员执行并要求影响分析、二次确认和审计。

条款或内容模块停用或废弃后的运行期影响已确认：已发布且已锁定该模块版本的模板仍按发布时锁定内容生成；模块停用或废弃只阻止后续新的模板发布候选引用。需要立即停止使用包含问题模块的已发布模板时，必须通过停用对应模板或发布版本来阻断生成和 API 调用。

结构化片段与母版样式映射已确认：v1 样式引用来自审核通过的母版样式目录；模板可引用命名样式，也可在白名单内配置有限直接格式；缺失、越界、不适用或无法稳定渲染的样式/直接格式阻断发布；母版样式目录变更不自动影响已发布模板。

Word/HTML 粘贴清洗规则已确认：只转换为 v1 受控节点和受控属性；简单表格和本地图片可转为组件草稿；高风险内容阻断；清洗后必须展示摘要并支持取消或撤销。

锚点绑定、渲染配置和预览产物边界已确认：模板只绑定已审核母版锚点目录中的稳定 `anchorId`，每个 `anchorId` 最多一个绑定且绑定一个有序结构化内容树；渲染配置采用发布锁定的受控 `renderProfile`；测试、审批和发布以最终 DOCX/PDF 渲染链路产生的预览产物和最终产物引用为权威证据，并通过并排对比视图解释差异。

## 一阶原则

### 1. 用户价值优先于平台治理

平台的核心价值不是拥有复杂 API 管理能力，而是让业务用户能稳定、低成本、低风险地生成正确金融信函。API、权限、审计、审批和发布门禁应服务于这一目标，不能反过来使模板创作变得难以使用。

### 2. 母版应是版式容器，不应承载大量业务内容

对于 100 页级复杂银行信函，母版如果承载大部分正文、条款和细粒度锚点，会造成维护灾难。推荐方向是：母版维护页眉页脚、版式样式、签章区、附件区和少量章节级锚点；正文内容由模板结构、条款片段、表格组件和规则编排生成。

### 3. 在线编辑应编辑结构化文档内容，而不是在线自由编辑整份 Word

模板编排人员需要在线维护章节、条款、变量、规则、表格和预览，但不应被要求在网页中自由排版一份完整 Word 文档。复杂页眉页脚、纸张、页边距、基础样式和签章区仍应主要由母版控制。

### 4. 富文本必须受控，不能等同于任意 HTML/CSS

DOCX 和浏览器 HTML/CSS 的排版模型不同。若以任意 HTML 到 DOCX 的黑盒转换作为核心能力，编号、缩进、分页、表格、字体和变量混排很容易失真。推荐方向是以受控结构化富文本模型作为源数据，再映射到 DOCX 段落、run、样式、编号和表格结构。

### 5. DOCX 保真是核心产品能力，不是渲染实现细节

用户是否信任产品，取决于生成 DOCX/PDF 是否稳定、可预览、可解释、可复测。富文本、编号、表格、分页、签章和 PDF 转换保真应进入产品验收和发布前检查，而不是留到实现末端处理。

### 6. 复杂度应留在可版本化、可复用、可测试的内容模块中

复杂信函的变化主要来自条款、章节、表格、变量和规则，而不是母版本身。系统应优先让这些内容模块可复用、可差异比较、可审批、可追溯，而不是把复杂性压到单个 DOCX 文件或单个模板页面。

### 7. 治理强度应与业务风险匹配

API 授权、幂等、审计和生命周期治理已经比较完整。后续需要避免继续堆叠外围治理而忽视创作可用性。新增治理应优先解决高风险内容问题，例如条款版本、富文本保真、发布前预览、差异摘要和模板调试。

## 当前文档审查结论

### 已经较强的部分

- API 路由、版本、幂等、错误模型、下载安全和 OpenAPI 契约已经较完整。
- API 管理配置与模板编排已经明确分离。
- 模板生命周期、发布版本锁定、测试审批和发布前门禁已经较完整。
- 审计、安全、AD Group、API 凭证和动态加密边界已经较清晰。

### 仍偏薄的部分

- 模板创作体验还停留在能力清单层面，缺少结构化编辑流程。
- 富文本只被列为锚点内容类型，尚未定义受控富文本模型。
- 条款片段尚未被建模为可复用、可版本化、可审批的内容资产。
- DOCX 保真、预览精度、编号稳定性和表格分页验收标准尚未定义。
- 信贷客户经理和模板编排人员的关键工作流仍在待确认问题中。

### 头重脚轻风险

如果继续优先深化 API 治理，而不补齐模板创作和渲染保真，平台可能变成“API 管理很强，但业务用户难以维护模板”的系统。对金融信函平台来说，这会偏离第一性原理：用户买账的是正确、稳定、可维护的信函生成能力。

## 推荐目标形态

### 文档资产分层

```text
母版
  负责 DOCX 基础版式、页眉页脚、样式、签章区、附件区和章节级锚点。

模板
  负责文档结构、章节顺序、锚点绑定、变量 Schema、规则和发布流程。

结构化文档片段
  负责富文本段落、变量 chip、条件块、循环块、编号层级和条款引用。

条款或内容模块
  负责可复用业务条款、标准段落、可选条款、适用范围和版本历史。

渲染配置或样式映射
  负责将结构化内容映射到 DOCX 样式、编号、表格、图片、签章和 PDF 转换。

API 调用
  负责运行时变量、输出格式、输出模式、加密参数和调用上下文。
```

### 100 页级信函的推荐维护方式

不推荐：

```text
一个巨大 DOCX 母版
  包含 100 页正文
  包含大量细粒度锚点
  每个条款和段落都在母版中维护
```

推荐：

```text
版式母版
  少量大区块锚点

模板结构树
  章节、附表、签章和附件顺序

结构化片段和条款模块
  条款内容、变量、编号、条件和循环

表格组件
  授信额度表、费用表、担保清单、条件先决清单

发布版本
  锁定一次完整组合
```

### 母版锚点粒度启发式

以下只是审查启发式，不是已确认限制：

| 锚点数量 | 审查判断 |
| --- | --- |
| 5 到 15 个 | 通常适合版式容器和章节级锚点。 |
| 20 到 40 个 | 需要检查是否开始把业务内容塞进母版。 |
| 50 个以上 | 高维护风险，通常应拆为结构化内容模块。 |
| 100 个以上 | 基本说明母版承担了过多业务模板职责。 |

## 结构化富文本方向

推荐将富文本能力定位为“结构化文档片段编辑器”，而不是普通网页富文本编辑器。

### v1 片段模型确认能力

| 能力 | 说明 |
| --- | --- |
| 块级结构 | 章节/标题、段落、列表、条件块、循环块、表格组件引用。 |
| 行内结构 | 文本片段、变量节点、强调、下划线、换行。 |
| 引用节点 | 条款或内容模块引用、图片引用、二维码/条码引用、签章/盖章引用、附件清单引用、样式引用。 |
| 编号模型 | 章节编号、条款编号和列表编号作为 v1 重点；更复杂编号细则仍需在编号模型和渲染验收中细化。 |
| 变量绑定 | 从模板变量 Schema 插入不可拆分变量节点。 |
| 规则绑定 | 对块或片段配置显示条件、循环数据源、校验约束。 |
| 样式映射 | 样式引用来自审核通过的母版样式目录；模板可选择命名样式，也可在白名单内配置有限直接格式。 |
| 校验输出 | 输出变量引用、样式引用、规则引用、校验结果和渲染风险。 |

### 样式映射确认

- v1 样式引用来自审核通过的 DOCX 母版样式目录。
- 母版样式目录需要标明可用样式名称、适用节点类型和渲染用途。
- 模板只能引用当前可访问、已审核通过母版提供的样式。
- 模板编排人员可选择命名样式，也可在平台白名单内配置有限直接格式。
- v1 有限直接格式仅包括字体、字号、文本颜色、行距、段前段后间距、首行缩进、左缩进和右缩进。
- 有限直接格式必须作为受控结构化属性保存，不得以任意 CSS、任意 Word 脏样式或脚本形式进入发布契约。
- 有限直接格式不得修改页眉页脚、页边距、纸张、全局版式、复杂分栏、浮动或绝对定位。
- 样式引用缺失、样式未通过母版审核、样式不在可访问母版样式目录内、样式不适用于节点类型、有限直接格式取值不在白名单内或无法稳定渲染，均阻断发布。
- 母版样式目录变更后，不自动影响已发布模板；已发布模板锁定发布时引用的样式引用和有限直接格式取值。
- 模板升级母版样式目录或调整样式映射时，必须执行影响分析，并重新经过测试、审批和发布流程。

### v1 不支持的自由能力

- 任意 HTML/CSS 到 DOCX 的无损转换。
- 脚本。
- 用户自由设置绝对定位、浮动或复杂分栏。
- 普通模板编排人员在线自由修改复杂页眉页脚、纸张、页边距和全局版式。
- 未经平台模型识别的 Word 脏样式直接进入发布版本。

### 粘贴处理确认

Word 或 HTML 粘贴内容必须先清洗并转换为受控结构化节点或受控结构化属性。不可识别内容不得作为原始格式直接进入发布契约。

已确认粘贴自动转换范围：

- 段落。
- 章节/标题。
- 列表。
- 文本片段。
- 强调。
- 下划线。
- 换行。
- 样式引用。
- 白名单内有限直接格式。

已确认组件草稿转换范围：

- 简单表格可转换为表格组件草稿。
- 本地图片可转换为图片引用草稿。

以下内容不得静默入模：

- 复杂表格或无法可靠识别结构的表格。
- 外链图片。
- 粘贴中疑似签章/盖章、二维码/条码或附件清单的内容；这些内容必须通过对应受控节点或组件插入。

以下内容阻断本次粘贴进入模板：

- 脚本。
- 宏。
- 嵌入对象。
- iframe。
- 外部资源。
- 绝对定位。
- 浮动。
- 复杂分栏。
- 未识别 Word 样式。
- 无法映射的变量。
- 无法映射的样式。
- 无法可靠识别的表格结构。

粘贴清洗完成后必须向用户展示清洗摘要，至少区分已转换内容、已删除内容、警告项和阻断项。用户必须可以取消本次粘贴，或在粘贴后撤销到粘贴前状态。粘贴清洗摘要和阻断项需要进入模板编辑或发布前检查的可追溯信息，但不得保存被清洗内容中的敏感明文或原始源内容。

## DOCX 保真审查维度

DOCX/PDF 保真的验收至少覆盖：

- 字体、字号、行距、缩进和段落样式。
- 多级编号、条款编号和条件隐藏后的编号重排。
- 表格列宽、表头、跨页、重复表头和单元格内容换行。
- 图片、二维码、签章、盖章和附件清单位置。
- 页眉页脚、页码、目录和附表编号。
- 变量格式化，例如金额、日期、百分比、枚举映射和本地化。
- DOCX 到 PDF 转换后版式一致性。
- 预览与最终下载文件的一致性边界。

## 发布前检查补充

以下阻断项已确认：

- 结构化富文本中存在不支持节点时阻断发布。
- 引用不存在的变量、样式、条款或表格组件时阻断发布。
- 引用不存在的母版锚点时阻断发布。
- 条件隐藏或循环渲染导致编号断裂且影响条款语义时阻断发布。
- 表格无法可靠渲染时阻断发布。
- 签章/盖章位置异常时阻断发布。
- PDF 转换失败时阻断发布。

以下警告项阈值和展示文案已确认：

- v1 不设置静默忽略阈值；任一已确认低风险保真问题都必须形成警告。
- 警告项不会仅因数量自动阻断发布；如果问题影响文档语义、合规结果、签章/盖章、二维码/条码、附件完整性、变量或引用合法性、关键表格可读性或条款编号语义，则应归为阻断项。
- `OPTIONAL_CONTENT_EMPTY`：可选章节、段落或表格因条件/循环为空，但已完整隐藏或配置空状态，且不影响编号和语义。
- `LOW_RISK_PAGINATION_DIFFERENCE`：预览、DOCX 和 PDF 之间存在不影响语义、签章、附件和表格可读性的低风险分页或换行差异。
- `LOW_RISK_TABLE_PAGE_BREAK`：表格跨页但表头、行内容和阅读顺序保持完整，且不影响关键金额或条款。
- `CONTROLLED_STYLE_FALLBACK`：使用已批准替代字体/样式或受控格式归一化，且不影响语义、页眉页脚或签章区。
- `IMAGE_SCALING_ADJUSTED`：普通图片为适配容器发生轻微等比缩放；二维码、条码、签章和盖章不适用图片缩放微调警告。
- 分页和换行警告按语义影响判定，不设置固定页数容忍。
- 警告展示必须包含稳定 `warningCode`、`messageKey`、影响位置摘要、检测结果摘要和处理建议；不得包含模板变量原值、粘贴原文、客户数据、完整请求体或生成文档敏感内容。
- 警告需要进入测试生成记录、审批材料、发布前检查清单、发布摘要、审计摘要和 API 生成成功响应。

以下复杂编号、表格组件、签章/盖章和 PDF 转换验收细则已确认：

- v1 复杂编号只支持平台或已审核母版样式目录确认的受控多级编号方案；条件块和循环块渲染后必须确定性重排编号，重复编号、跳号、条款编号语义变化或交叉引用失效均阻断发布。
- v1 表格组件采用结构化表格组件模型，支持列 schema、表头、跨页重复表头、循环行、简单合计/页脚和受控列宽；嵌套表、浮动表、绝对定位表、无法稳定跨页的合并单元格或无法保持关键金额/条款可读性的表格均阻断发布。
- v1 签章/盖章仅确认受控占位和印章/签名图片资产，不包含密码学电子签名、电子印章或验签能力；签章/盖章必须位于授权区域内，不得重叠、裁切、越界或在 DOCX/PDF 中不可见，否则阻断发布。
- PDF 转换验收采用语义保真优先基线；不得出现缺页、异常空白页、内容裁切、关键内容丢失、签章/盖章不可见、二维码/条码不可读、关键表格不可读或附件引用丢失。低风险视觉差异按已确认警告项处理。
- 编号、表格、签章/盖章和 PDF 转换验收结果必须进入测试生成记录、审批材料、发布前检查清单、发布摘要和审计摘要；发布前检查存在阻断项时不得发布。
- 已发布模板运行期生成对外只返回生成成功警告或生成失败错误；API 契约和响应不得暴露内部渲染诊断明文、模板变量原值、客户数据或完整生成内容。

结构化富文本和片段编辑能力的 v1 复用范围已确认：仅用于文档生成主线，包括模板正文、结构化文档片段、条款或内容模块，以及已确认的受控表格、图片、二维码/条码、签章/盖章和附件清单节点。v1 不扩展到通知模板、审批意见模板、测试意见模板或其他非文档生成业务文本场景；未来如需复用，必须单独确认对象模型、权限、生命周期、审批、审计和输出边界。

锚点绑定、渲染配置和预览产物细则已确认：

- v1 模板绑定母版锚点必须来自已审核母版锚点目录中的稳定 `anchorId`，并在发布版本中锁定母版版本和锚点目录版本；不得依赖 Word 名称、文档位置或模糊匹配作为发布契约。
- 每个 `anchorId` 在同一模板发布候选中最多一个绑定；绑定对象承载一个有序结构化内容树，内容类型必须符合锚点允许范围，绑定校验状态至少包括 `VALID`、`MISSING_ANCHOR`、`DUPLICATE_BINDING` 和 `INCOMPATIBLE_CONTENT_TYPE`；后三者均属于发布阻断项。
- v1 渲染配置采用发布锁定的受控 `renderProfile`，覆盖样式、编号、表格分页、图片缩放、PDF 转换和保真策略；API 调用方不得传入任意渲染参数覆盖模板发布配置。
- v1 生成预览产物采用最终 DOCX/PDF 渲染链路产生的可追溯预览记录，至少包含 `previewId`、`testDataSetId`、`variablesHash`、`outputFormat`、`renderProfileVersion`、`status`、`documentId` 或预览引用、最终产物引用、警告摘要、阻断项摘要、对比摘要、创建时间、更新时间和非敏感摘要；预览状态集合为 `ACCEPTED`、`PROCESSING`、`SUCCEEDED`、`FAILED` 和 `EXPIRED`。
- 编辑态快速预览只用于即时反馈，不作为测试、审批或发布权威证据；测试、审批和发布摘要按输出策略锁定最终 DOCX/PDF 产物引用、预览记录引用、对比摘要和非敏感摘要；预览文件、最终产物和并排对比视图按权限受控查看，审计摘要不得保存模板变量原值、客户数据、完整请求体或完整生成内容。

## 已确认关键结论

1. 母版主要作为版式容器和大区块锚点载体，不承载 100 页级信函正文内容。
2. 模板编排采用结构化编辑，而不是在线自由编辑整份 Word。
3. 富文本源数据采用受控结构化模型，而不是任意 HTML/CSS。
4. 条款片段升级为可复用、可版本化、可审批的条款或内容模块。
5. DOCX/PDF 预览和渲染保真采用阻断项与警告项分级。
6. v1 采用保守结构化富文本节点矩阵，支持章节/标题、段落、列表、条件块、循环块、表格组件引用、文本片段、变量节点、强调、下划线、换行、条款或内容模块引用、图片/二维码/签章/附件引用和样式引用。
7. v1 不支持任意 HTML/CSS、脚本、绝对定位、浮动、复杂分栏、在线任意修改全局版式或未识别 Word 脏样式进入发布版本。
8. Word 或 HTML 粘贴内容必须清洗并转换为受控结构化节点。
9. 不支持节点、关键引用缺失、影响条款语义的编号断裂、表格无法可靠渲染、签章/盖章位置异常和 PDF 转换失败均属于发布阻断项。
10. 条款或内容模块采用独立版本审批生命周期；发布模板锁定具体已批准模块版本；模块新版本不自动影响已发布模板；停用、恢复和废弃由管理员治理。
11. 结构化片段与母版样式映射采用审核通过的母版样式目录，允许白名单内有限直接格式，越界或无法稳定渲染时阻断发布。
12. Word/HTML 粘贴只转换为受控节点或受控属性，复杂或高风险内容阻断，清洗后必须展示摘要并支持取消或撤销。
13. 条款或内容模块停用或废弃后，已发布且锁定该模块版本的模板仍按锁定内容生成；需要阻断生成时，通过停用模板或发布版本执行。
14. DOCX/PDF 保真警告采用五类稳定 warningCode；任一低风险项都进入测试、审批、发布、审计和 API 成功响应，但不因数量自动阻断发布。
15. 复杂编号、表格组件、签章/盖章和 PDF 转换采用受控验收边界；不稳定或影响语义、合规、关键可读性和签章区域的结果阻断发布。
16. 结构化富文本和片段编辑能力 v1 仅复用于文档生成主线，不扩展到通知、审批意见、测试意见或其他非文档生成业务文本。
17. 锚点绑定采用稳定 `anchorId` 和发布锁定的母版/锚点目录版本；渲染配置采用发布锁定的受控 `renderProfile`；最终 DOCX/PDF 渲染链路产生的预览产物、最终产物引用和并排对比摘要作为可追溯记录进入测试、审批和发布证据。
